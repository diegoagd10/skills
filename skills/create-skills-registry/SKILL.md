---
name: create-skills-registry
description: Scan every known skill directory (global + project) and render a compact index at
  .atl/registry.md — one row per skill with a short summary and its exact SKILL.md path — so an
  orchestrator can pick which skill to delegate and inject its path into a sub-agent. Use after
  installing, creating, moving, or renaming skills, or when "create/refresh the skill registry".
---

Build the **delegator's index** of skills. The registry lets an orchestrator know what skills
exist and where to find them, without holding every full `SKILL.md` in context. The orchestrator
reads the **Summary** to choose; it passes the **Path** to the sub-agent (never the summary —
`SKILL.md` stays the source of truth). See `skills/_shared/skill-resolver.md` for consumption.

## Scan sources (skip any that don't exist)
Global: `~/.claude/skills`, `~/.agents/skills`, `~/.config/opencode/skills`, `~/.codex/skills`,
`~/.gemini/skills`, `~/.copilot/skills`, `~/.pi/agent/skills`, `~/.codeium/windsurf/skills`.
Project (cwd): `.claude/skills`, `.agents/skills`, `skills/`.

## Rules
- **Resolve symlinks** (`realpath`) and **dedup by skill name** — `.claude/skills`,
  `.agents/skills`, and `skills/` usually resolve to the same files; list each skill once.
  On a name clash across sources, **prefer the project-level** copy.
- **Skip** `_shared`, `create-skills-registry` itself, and any directory without a `SKILL.md`.
- The **Summary** is a condensed ≤12-word phrase derived from the skill's `description`
  frontmatter (what it does + trigger) — short, to keep the orchestrator's context small. It is
  a selection aid, NOT a replacement for the skill; do not inject summaries into sub-agents.
- The registry is **local**: write `.atl/registry.md` (overwrite if present) and ensure `.atl/`
  is in `.gitignore`.

## Steps
1. For each existing source, glob `*/SKILL.md`; read only the frontmatter (`name`, `description`).
2. Dedup by name (prefer project); build a short summary per skill.
3. Write `.atl/registry.md` (overwrite) using the template below.
4. Persist to engram for compaction-safe recovery:
   `mem_save(title/topic_key: "skill-registry", type: "config", project, capture_prompt: false,
   content: "{the rendered registry}")`.
5. Report: registry path, skill count, sources scanned, any skipped/duplicate names.

## `.atl/registry.md` template
```markdown
# Skill Registry — {project}
<!-- Auto-generated by create-skills-registry. Re-run to refresh. -->
Last updated: {YYYY-MM-DD}

## Contract
Delegator-use index. To delegate, match the task against **Summary**, then inject the matching
**Path** values under `## Skills to load before work` in the sub-agent prompt. The sub-agent
reads those exact `SKILL.md` files before working. `SKILL.md` is the source of truth — never
pass the summary in place of the file.

## Skills
| Skill | Summary | Path |
| --- | --- | --- |
| `slice-runner` | Orchestrate a PRD's slices end-to-end via sub-agents | `/abs/.../skills/slice-runner/SKILL.md` |
| ... | ... | ... |

## Sources scanned
- /abs/path/that/existed
- ...
```
